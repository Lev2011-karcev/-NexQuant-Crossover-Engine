# 💥 Mini Quant Backtesting Engine (C++)

## 🧠 Общая идея
Проект представляет собой **event-driven quant trading engine** (событийно-ориентированный движок для квантового трейдинга), который выполняет следующие задачи:
* Обрабатывает потоковые рыночные данные (market data)
* Рассчитывает технические индикаторы в реальном времени
* Генерирует торговые сигналы на основе скоринга
* Симулирует исполнение сделок
* Проводит бэктестинг стратегий на исторических данных
* Оценивает финансовую доходность и риски

---

## ⚙️ Архитектура системы
Потоковая обработка данных организована в виде цепочки:

```text
Market Data → Event Engine → Indicator Engine → Strategy Engine → Execution Simulator → Backtest Report
```

---

## 📦 Основные модули

### 1. 📥 Data Feed (Вход данных)
Отвечает за трансляцию потока цен. 
* **Пример входящего потока:** `AAPL 190` → `AAPL 191` → `AAPL 189`
* **Базовая структура данных:**
```cpp
struct Tick {
    std::string symbol;
    double price;
};
```

### 2. ⚡ Event Engine (Движок событий)
Каждое новое значение цены инициализирует цепочку событий:
```text
PRICE_EVENT → update state → update indicators → generate signal
```
* **Задача:** централизовать обработку данных и мгновенно обновлять состояние всей системы при каждом тике.

### 3. 📊 Indicator Engine (Вычисление индикаторов)

* **📈 SMA (Simple Moving Average)**
  \[\text{SMA}_n = \frac{1}{n}\sum_{i=0}^{n-1} P_i\]
  * *Реализация:* скользящее окно (sliding window) + кумулятивная сумма.
* **⚡ EMA (Exponential Moving Average)**
  \[\text{EMA}_t = \alpha P_t + (1-\alpha) \text{EMA}_{t-1}, \quad \alpha = \frac{2}{n+1}\]
  * *Реализация:* рекурсивный расчет, не требующий хранения всей истории цен.
* **📉 RSI (Relative Strength Index)**
  \[\text{RSI} = 100 - \frac{100}{1 + \text{RS}}, \quad \text{RS} = \frac{\text{AvgGain}}{\text{AvgLoss}}\]
  * *Реализация:* скользящее окно по изменениям цены (diffs).
* **🌪 Volatility (Риск-метрика)**
  \[\sigma = \sqrt{\frac{1}{n}\sum (P_i - \bar{P})^2}\]
  * *Реализация:* окно цен + динамическое обновление статистических моментов.

> 💡 **Важно: Оптимизация Sliding Window**
> Используется в **SMA**, **RSI** и **Volatility**. 
> Вместо полного пересчета истории мы храним только последние \(N\) значений в `std::deque<double> window`. При поступлении нового тика мы добавляем его в конец, удаляем старое значение спереди и атомарно корректируем метрики.

### 4. 🧠 Strategy Engine (Генерация сигналов)
Каждый индикатор отдает свой независимый «голос»:
* `BUY` = \(+1\)
* `SELL` = \(-1\)
* `NEUTRAL` = \(0\)

**Пример голосования:**
* SMA → `BUY` (\(+1\))
* EMA → `BUY` (\(+1\))
* RSI → `SELL` (\(-1\))
* Momentum → `BUY` (\(+1\))

\[\text{Score} = \sum \text{signals} = 1 + 1 - 1 + 1 = 2\]

**Принятие решения:**
* \(\text{Score} \ge 2 \rightarrow\) **BUY**
* \(\text{Score} \le -2 \rightarrow\) **SELL**
* Иначе \(\rightarrow\) **HOLD**

### 5. 💰 Execution Simulator (Симулятор биржи)
Виртуально исполняет ордера:
* Фиксирует покупку / продажу активов
* Ведет учет открытых позиций
* Рассчитывает текущий PnL (чистую прибыль и убыток)

### 6. 📉 Backtesting Engine (Ключевой модуль)
Отвечает за историческое моделирование.
* **Вход (Input):** Исторические массивы цен (historical prices).
* **Выход (Output):** Метрики эффективности стратегии (performance metrics):
  * Total Return (Общая доходность)
  * Win Rate (Процент прибыльных сделок)
  * Max Drawdown (Максимальная просадка капитала)
  * Number of Trades (Количество совершенных сделок)

### 7. ⚠️ Risk Engine (Анализ рисков)
Оценивает качество поступающих сигналов на лету:
* Анализирует текущую волатильность рынка (`Volatility`)
* Выявляет конфликтность индикаторов (когда разные метрики противоречат друг другу)
* Контролирует текущий уровень просадки (`Drawdown`)

## ⚙️ Event Flow (Жизненный цикл событии)

```text
  PRICE IN (Поступление цены)
      ↓
 update state (Обновление состояния)
      ↓
 update indicators (Пересчет индикаторов)
      ↓
strategy evaluation (Оценка стратегии)
      ↓
 signal output (Выдача торгового сигнала)
```

```text
SIGNAL: BUY (confidence 78%, risk HIGH)
```

## 🚀 План разработки MVP

1. **База:** Настройка `DataFeed`, парсинг и хранение входящих цен.
2. **Индикаторы:** Написание математических классов для `SMA`, `EMA`, `RSI`.
3. **Стратегия:** Реализация логики голосования (voting system).
4. **Backtest:** Запуск прогона исторических данных через движок событий.
5. **Метрики:** Расчет финальных показателей `Profit`, `Drawdown` и `WinRate`.

## 🔥 Что делает проект «сильным» для портфолио
* Архитектура, полностью построенная на событиях (**Event-driven architecture**).
* Потоковая обработка данных без загрузки всей истории в память (**Streaming data processing**).
* Расчет индикаторов в реальном времени с оптимизацией по скорости (**Real-time indicators**).
* Полноценный модуль исторического тестирования и оценки рисков.
